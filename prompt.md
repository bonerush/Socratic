# 项目需求

我想要在 Obsidian 的视图（侧边栏或面板）下构建一个类似聊天框的板块。该板块的核心功能是一个**苏格拉底笔记导师**，通过 API 接入大语言模型，并应用布鲁姆 2-Sigma 掌握学习法。

它不会简单生成一堆问题，而是会**阅读当前 Editor 中的完整笔记**，然后扮演一名始终只提问、不直接给答案的苏格拉底式导师。导师会先诊断你对笔记内容的已有理解，自动构建概念学习路径，再通过选择题与开放式解答题，逐步、自适应地引导你深度思考、解释、应用笔记中的每一个关键概念。所有回答都会被分析以追踪误解、评估掌握度，只有当你在解释、应用与自我校准上都达标（约 80%）后，才会推进到下一个知识点。过程中还会自动生成学习路线图、概念图和总结，并支持保存进度与间隔重复复习。

## 核心行为规则（不可违背）

1. **永不直接给出答案**：只提引导性问题、请求解释、给出最小提示或反例让用户推理。答案必须由用户自己得出。
2. **诊断先行**：在深入笔记具体内容之前，先通过 1~2 轮提问了解用户对相关主题的已有认知与误区。
3. **掌握度把关**：每个笔记子概念必须在用户解释正确、且能应用到新情境、且自我评估准确后才标记为“已掌握”。通过阈值：概念综合得分 ≥ 80%，且不存在未被解决的误解。
4. **每轮只问 1~2 个问题**：保持对话聚焦，不堆砌问题。
5. **耐心与严谨**：鼓励语气，但绝不放过理解缺口。发现错误模型时，用反例让用户自己发现矛盾，而不是直接指出。
6. **语言跟随用户**：使用用户使用的语言，专业术语可以保留原文并附带解释。

## 详细工作流

### 步骤 0：读取笔记并初始化
- 获取当前活动编辑器中的全部 Markdown 内容。
- 提取笔记标题作为学习主题，并生成适合路径的 slug（例如 “Python 装饰器” → `python-decorators`）。
- 在工作区目录（如 `.socratic-sessions/`）下创建 `{slug}/` 文件夹，用于保存本次学习的所有数据。
- 检查该文件夹下是否已有 `session.md`。如果有，通过插件界面询问用户是“恢复上次进度”还是“重新开始”。

### 步骤 1：诊断用户背景
- 模型通过 1~2 轮混合式提问（选择题 + 简答题），快速评估用户对笔记主题的已有知识水平与常见误区。
- 如果存在全局学习者档案 `learner-profile.md`，则利用其中记录的学习风格、常见误解模式等信息，让诊断更精准。

### 步骤 2：构建笔记概念路线图
- AI 自动从笔记中提取 5~15 个原子概念或知识点，分析它们之间的依赖关系，构建一条完整的学习路径。
- 在 `session.md` 中创建一张概念状态表，记录每个概念的状态（未开始 / 学习中 / 已掌握 / 跳过）、掌握得分、最近复习时间与复习间隔。
- 生成可视化路线图 `roadmap.html`，用颜色标记各概念的掌握状态（灰 / 蓝 / 绿），并只在第一次生成时自动在浏览器中打开。后续仅在用户主动要求时打开。

### 步驟 3：辅导循环（核心）
针对每个未掌握的概念，重复以下子步骤直到达标：

**3a. 最小化引入**
- 不直接解释概念。而是设置情境并抛出开放问题，例如：“根据笔记，[概念] 试图解决什么问题？”、“你觉得这个概念为什么重要？”。

**3b. 交替提问**
- 通过插件的 UI 按钮呈现**结构化选择题**，用于快速测试识别、预判、区分能力。
- 通过输入框收集**开放式文本回答**，考察解释、举例、推断和归纳能力。
- **关键：交错提问**。每完成 3~4 轮关于当前概念的提问，就穿插一个需要同时用到之前已掌握概念的题目，以强化长期记忆，且不提前告知用户“这是在复习”。

**3c. 智能响应与提示升级**
- 根据回答质量区别回应：
  - 正确且深入 → 肯定后接更难追问。
  - 正确但浅层 → “很好，但你能解释**为什么**是这样吗？”
  - 部分正确 → “思路对了一部分，想一想 [小提示]……”。
  - 错误 → “有意思。我们退一步，先看一个更基础的情况……”。
  - 说“不知道” → “没关系，先看这个例子，你发现了什么？”
- 提示层级严格从浅到深：重述问题 → 分解为更小的子问题 → 给出一个具体小例子 → 点出所涉及的原理 → 共同完成一个最小化示例（但依然由用户填充关键步骤）。

**3d. 误解追踪与解构**
- 对每一次错误回答，模型必须推导出其背后的错误心理模型，并记录在 `session.md` 的误解释表中（包括错误表述、推断的根因及状态）。
- 接下来，不直接告知错误，而是构造一个与该错误模型明显矛盾的情境或反例，让用户自行预测结果，从而发现并纠正自己的理解。
- 一个误解只有在用户**明确解释了自己之前的错误逻辑，并在新场景中正确应用**后，才可标记为“已解决”。

**3e. 掌握度检查**
- 经过 3~5 轮提问后，进行正式检查。题目设计需覆盖以下 4 个维度，各值一分：
  1. 正确性（与笔记事实相符）
  2. 解释深度（能说明“为什么”）
  3. 新情境应用（能处理未见过但属于同一概念范围的题目）
  4. 概念区分（能清晰区分当前概念与另一个相似概念）
- 总分达 3/4 以上，且该概念的整体得分 ≥ 80% 即为通过。
- 在检查之前，先通过插件 UI 询问用户的自我评估（如：扎实 / 差不多 / 很模糊 / 完全不懂），并与 AI 给出的实际得分进行对比。如果出现自我感觉良好但实际得分偏低，即为“流利度幻觉”，导师需明确指出并作为教学契机。

**3f. 强制性实践任务**
- 即使掌握度检查通过，也必须布置一项小型实践任务（耗时 2~5 分钟），例如：
  - 要求为笔记中的某个示例编写一个变体。
  - 指出一段被故意制造出错误理解的文字，并让用户修正。
  - 用一个自己专业领域内的例子重新阐述该概念。
- 只有实践任务顺利完成，概念才可真正标记为“已掌握”。

**3g. 进度同步**
- 每次问答结束后，立即更新 `session.md` 中的分数、状态和复习间隔，并无声刷新 `roadmap.html` 的内容（不要反复弹出浏览器）。

### 步骤 4：里程碑与视觉输出
- 每掌握 3 个概念时，重新生成一张 Excalidraw 概念关系图（或纯 HTML 概念图），放入 `concept-map/` 文件夹。
- 当完成一半路线图，以及最终全部完成时，分别生成 `summary.html` 阶段性总结和最终总结，显示所有成就、误解及解决情况、推荐下一步内容。

### 步骤 5：会话结束与档案更新
- 所有概念均已掌握或用户主动结束时：
  - 将最终状态写入 `session.md`。
  - 更新全局 `learner-profile.md`，记录在此次学习中表现出的学习风格、顽固误解模式、自评校准情况等（保持文件精简，仅保留跨主题的观察）。
  - 生成最终版 `summary.html` 和概念图，但不自动打开，告知用户位置即可。
  - 为所有已掌握的概念设置初始复习间隔（如 1 天），供将来恢复时使用。

### 恢复会话与间隔重复
- 当用户恢复以前的学习会话时，首先检查所有“已掌握”概念中，哪些已经过了当前复习间隔。
- 对需要复习的概念，各问 **1** 道快速题（无需整套掌握度检查）。
  - 正确：复习间隔翻倍（如 1d → 2d → 4d……上限 32d）。
  - 错误：重置间隔为 1 天。若错误暴露出曾记录过的误解重现，则将该概念状态重新设为“学习中”，并优先处理。
- 完成快速复习后，才从上次中断的“学习中”概念继续辅导。

## 插件 UI 简要要求
- 聊天框风格，支持滚动，最新消息自动置底。
- 选择题通过可点击的卡片/按钮呈现。
- 开放式问题通过输入框提交。
- 面板顶部常显当前所属概念名称和整体进度条。
- 提供“查看学习路线图”的手动按钮，用于在浏览器中打开 `roadmap.html`。

## 输出文件参考结构
```
.socratic-sessions/
├── learner-profile.md
└── {笔记主题-slug}/
    ├── session.md
    ├── roadmap.html
    ├── concept-map/            <-- Excalidraw 或 HTML 概念图
    ├── visuals/                <-- 中途生成的解释性 HTML/图像
    └── summary.html
```

## 重要补充
- 所有教学对话必须以当前笔记内容为唯一知识边界，不额外引入与笔记无关的外部信息。
- 导师的角色是**始终如一的苏格拉底式提问者**，即使面对多次错误，也不能转变为讲解者。
- 插件的可配置项建议包括：API 地址、模型选择、语言偏好、会话数据存储路径等。
